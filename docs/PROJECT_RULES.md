# PROJECT RULES

## Primary Goal

Get MockupExpo paying customers as fast as possible.

## Constraints

- Minimize spending.
- No new subscriptions unless approved.
- Minimize founder manual work.
- Prefer automation over manual processes.

## Pre-Implementation Checklist

Complete all 8 items in writing **before any coding**. Do not implement until every item is answered.

1. **State facts.** What is known from evidence, data, or direct observation?
2. **State assumptions.** What are you inferring or taking on faith?
3. **State confidence (High / Medium / Low).** How sure are you that this approach will work?
4. **Explain how this helps get MockupExpo customers.** Tie the work directly to revenue or customer acquisition.
5. **Explain founder time required.** Estimate manual effort; if more than 30 minutes, justify why.
6. **Explain cost.** Money, subscriptions, API usage, or other spend — including zero-cost if applicable.
7. **Explain how success will be proven.** What end-to-end evidence will demonstrate this works?
8. **What could make this recommendation wrong?** List the top 3 failure modes.

**Gate rule:** Do not implement until all 8 are answered. Do not declare complete without end-to-end evidence.

## Decision Rules

- Verify assumptions before implementation.
- Evidence before conclusions.
- Do not declare complete without proof.
- Show blockers early.
- Challenge proposed solutions before building.

## Communication Rules

- Facts first.
- Assumptions separately.
- Recommendations last.
- State confidence: High / Medium / Low.
- If a task requires more than 30 minutes of founder work, justify why.

## Development Rules

- Reuse existing code before creating new code.
- Prefer the smallest working solution.
- No redesign unless current approach is proven inadequate.
- Focus on revenue impact before technical elegance.

## Completion Rules

Nothing is complete until it is demonstrated working end-to-end with evidence. The Pre-Implementation Checklist gate rule applies: no work is done until all 8 items are answered, and no work is closed until success is proven with real evidence.
